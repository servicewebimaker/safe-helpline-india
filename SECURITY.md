# Security Policy

Safe Helpline India is an information toolkit. Security and privacy issues are treated seriously because people may use the project in sensitive situations.

## Reporting a Vulnerability

Please do not report vulnerabilities in a public issue.

Use GitHub private vulnerability reporting after the repository is published. If private reporting is not available, contact the repository owner through a private channel listed on the repository profile.

Include:

- A clear description of the issue.
- Steps to reproduce.
- Affected package or app.
- Impact and suggested mitigation, if known.

## Scope

In scope:

- Web app security issues.
- Android app privacy or permission regressions.
- Widget injection or isolation issues.
- Data validation flaws that could allow unsafe entries.
- Documentation that could cause unsafe operational assumptions.

Out of scope:

- Vulnerabilities in third-party services linked as public sources.
- Claims about helpline operations outside the project code and data.
- Social engineering against maintainers or service operators.

## Privacy Regressions

Please report any change that adds tracking, cookies, analytics, unnecessary permissions, automatic calls, automatic messages, contact collection, or location collection.
