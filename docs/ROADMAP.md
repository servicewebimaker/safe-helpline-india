# Roadmap

This roadmap is intentionally practical and verification-first.

## Near Term

- Add a pull request checklist for data changes.
- Add automated checks that compare `data/helplines.json` with the core package data.
- Add Playwright accessibility checks for the web app.
- Add Android unit tests for repository and search behavior.
- Add screenshots to documentation after the UI is visually reviewed.

## Medium Term

- Add localization support for Hindi and additional Indian languages.
- Add a static PWA build for offline web usage.
- Add maintainer review templates for public safety data.
- Add a signed release process for the Android app.
- Expand source metadata when official pages provide updated details.

## Long Term

- Include state-wise helplines only when each entry has official or public source verification.
- Build a community review process for source freshness.
- Add optional import/export tooling for downstream civic-tech projects.
- Publish the package and widget with versioned release notes.

No roadmap item should weaken the privacy policy, add tracking, or bypass data verification.
